What it does

A gauge for your Claude limits.

Server-side truth, fetched live from Anthropic. Not an estimate from local logs, the real numbers Claude uses to decide your limits.

A sparkle with moods

The mascot breathes when you're idle, plays while a window is open, and goes frantic as you near a limit. You feel your usage before you read it.

Start a window in one tap

No open session? Tap Start Window and Claudometer fires a tiny ping to begin your 5-hour clock, no terminal required.

Private by design

Your token never leaves your Mac. Claudometer reads your local Claude login and talks only to Anthropic. Nothing is sent to us, ever.

Connect your Claude

Up and running in three steps.

Claudometer shows your data by reading the same Claude Code login you already use. If you have Claude Code, you're basically done.

STEP 01

Install Claude Code

The free CLI from Anthropic. Skip this if you already use it.

$ npm i -g @anthropic-ai/claude-code
STEP 02

Sign in once

Run Claude Code and log in with your Claude account. That stores the login Claudometer reads.

$ claude
STEP 03

Open Claudometer

Launch the app. Your sparkle appears in the menu bar with live usage. First launch only, macOS asks you to confirm: open System Settings → Privacy & Security and click Open Anyway.

Questions

Good to know.

Is it free?
Yes. Claudometer is free and open source. You only need an active Claude Max plan to have usage worth watching.
Does it use up my Claude tokens?
No. Reading your usage costs nothing. The optional Start Window button sends one tiny haiku ping to open a session, and that's the only time it touches your quota, only when you ask.
Why does it need macOS 26?
The interface is built on Tahoe's Liquid Glass. That material only exists on macOS 26 and newer, on Apple Silicon.
Is my Claude data safe?
Your token stays on your Mac and is used only to call Anthropic's own usage API. Claudometer has no server and collects nothing. The source is public so you can verify it.
How do updates work?
The app checks for a newer version on launch and from its menu. When one is out, it shows "Update available" and links you here to download it.
The first launch shows a warning?
During early access the build isn't notarized yet, so macOS blocks the first launch. The old right-click → Open shortcut no longer works on macOS Tahoe, so instead: double-click Claudometer, click Done, then open System Settings → Privacy & Security and click Open Anyway next to Claudometer. Confirm once and it launches every time after. Signed builds are coming.
